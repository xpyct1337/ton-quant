# Промпт: собрать список Telegram-каналов для social.py

Скопируй текст ниже в любой LLM с доступом к вебу (Claude, ChatGPT, Perplexity).
Результат вставь в `data/social_channels.json` (поле `channels`).

---

Найди публичные Telegram-каналы про трейдинг мемкоинов на блокчейне TON и выдай
их в виде JSON-списка слагов.

КОНТЕКСТ. Слаги пойдут в автоматический сборщик, который раз в день читает
`https://t.me/s/<slug>` (публичное веб-превью последних ~20 сообщений, без
авторизации) и считает упоминания тикеров в форме кэштегов ($REDO, #UTYA).
Ценность канала = насколько часто в нём кэштегами обсуждают конкретные
TON-жетоны (mention velocity — опережающий сигнал внимания).

КРИТЕРИИ ВКЛЮЧЕНИЯ (все обязательны):
1. Тематика: колл-каналы / сигналы / трейдинг / обзоры мемкоинов именно на TON
   (jetton'ы: NOT, REDO, UTYA, DOGS, HMSTR и т.п.). Не общекриптовые.
2. Публичный канал с работающим веб-превью: страница `https://t.me/s/<slug>`
   открывается и показывает сообщения (некоторые каналы отключают превью — такие
   не подходят). ПРОВЕРЬ каждый слаг фактическим открытием этой страницы.
3. Живой: последнее сообщение не старше 7 дней, постит хотя бы несколько раз в
   неделю.
4. В сообщениях реально встречаются тикеры/кэштеги токенов, а не только общие
   новости.

ИСКЛЮЧИТЬ: официальные новостные каналы экосистемы (toncoin, ton_blockchain и
т.п. — они не кэштегают мемкоины), каналы-агрегаторы аирдропов без трейдинга,
мёртвые/переименованные, приватные (t.me/+...), чаты-группы (нужны каналы).

Язык каналов любой (русский/английский), 10–20 штук, отсортируй по
предполагаемой полезности (самые кэштеговые — первыми).

ФОРМАТ ОТВЕТА — строго один JSON-объект, без пояснений вокруг:

{
  "channels": ["slug1", "slug2", "..."],
  "rejected": [{"slug": "...", "reason": "почему не подошёл"}]
}

`slug` — это часть URL после t.me/, без @ и без https://.

---

После вставки в `data/social_channels.json` (заменить массив `channels`,
поле `note` можно оставить) — закоммить/задеплой; сборщик подхватит на
следующем прогоне, mention velocity появится через два дня данных.
