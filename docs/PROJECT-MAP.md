# TON Quant — карта проекта

*Живой документ. Обновлять при значимых изменениях архитектуры/данных/страниц —
не при каждом коммите. Дата последнего полного ресёрча: 2026-07-07. История решений
и «почему» — в `ROADMAP-v3.md` (актуальное) и `ROADMAP-alpha-archive.md` (архив).
Спек деска — `docs/V3.0-AI-SMART-MONEY-DESK.md`.*

Назначение файла: одно место, где написано **что есть в проекте, зачем и как это
читать** — простым языком, без предположения, что читатель помнит контекст прошлых
сессий. Ссылки `[[...]]` — не работают как гиперссылки, это просто метки понятий
для перекрёстных упоминаний внутри документа.

## 1. Что это

TON Quant — аналитика экосистемы TON: рынки, скринер токенов, глубокие карточки
токенов, ростер «умных денег» (smart-money wallets), paper-боты (симуляция
торговли без реальных денег), forward-test кросс-секционного моментума, и
**AI Smart-Money Desk** — слой локальных LLM-агентов, который оценивает риск
манипуляции и решает, можно ли копировать конкретный кошелёк.

Не финансовый совет — это исследовательский/аналитический инструмент.

## 2. Архитектура — три слоя

```
[Сбор]     GitHub Actions (по расписанию) → пишет data/*.json в репо
[Деск]     локальный Mac (демон 24/7)      → читает data/, считает фичи,
                                              гоняет LLM-агентов, пишет data/desk/*.json
[Витрина]  SvelteKit статика (app/)        → читает всё выше через raw.githubusercontent,
                                              рисует UI, часть страниц дозапрашивает live-данные
```

Слои общаются **только через файлы в git** — ни у сайта, ни у деска нет бэкенда.
Сайт — статика на GitHub Pages (`deploy.yml`, билд при пуше в `main`).

## 3. Как читать сайт — карта страниц

Каждая страница — `app/src/routes/<name>/+page.svelte`, данные тянет через
`$lib/data.js` (raw.githubusercontent) и/или напрямую с публичных API
(TONAPI, DexScreener, STON.fi, GeckoTerminal, Hyperliquid).

| Страница | Что показывает | Кому полезна | Источник данных |
|---|---|---|---|
| `/` (Markets) | Карта рынка (treemap), лента агентных сигналов, календарь анплоков, таблица всех jetton'ов | Обзор дня в 30 секунд | `loadAll()`, `loadSignals()` |
| `/brief` | **Дневной срез** — что изменилось за сутки по всем разделам сразу (см. §7) | Быстрый апдейт после отсутствия | все лоадеры + дневные диффы |
| `/analytics` | Конфигурируемая панель из ~14 виджетов (корреляции, RSI/MACD, risk-return, wash-detector, rug radar…) | Глубокий технический анализ | `loadAll(40)` + `$lib/widgets.js` |
| `/screener` | Широкий список всех обнаруженных токенов (не только отслеживаемых) | Поиск новых токенов | `loadUniverse()` |
| `/arb` | Разброс цены токена между пулами/DEX | Арбитраж | `loadAll()` + `$lib/arb.js` |
| `/perps` | Перпетуалы Hyperliquid: funding, OI, эвристический long/short bias, фокус на GRAM (экс-TON) | Перп-трейдинг | live Hyperliquid API, фолбэк `perp_markets.json` |
| `/tgsig` | Лента разобранных сигналов из Telegram (`@perptools_ai_bot`, `dexnewtoken`) | Сырой фид внешних сигналов | `perp_signals.json`, `dex_signals.json` |
| `/compare` | Сравнение токенов бок о бок | Выбор между похожими токенами | `loadUniverse()` + `loadCompare()` |
| `/portfolio` | Баланс любого TON-кошелька | Проверить чей-то (свой) портфель | live TONAPI, без данных репо |
| `/wallets` | Ростер smart-money кошельков + риск-значок и обоснование деска | Кого копировать | `loadWallets()` + `loadDeskStatus()` |
| `/xsmom` | Live forward-test кросс-секционной momentum-стратегии (перпы) | Проверка модельной стратегии | `loadXsForward()` |
| `/paper` | Симулированные портфели (cons/aggr/alt) — эквити, позиции, скоринг сигналов | Проверка «а сигналы вообще работают?» | `loadPaper()`, `loadRegimeBench()` |
| `/desk` | Витрина AI Desk: вердикты, калибровка, факторы ресёрчера, копи-трейд пруф | Доверять ли риск-оценкам деска | `loadDeskStatus/Calibration/Factors/Copytrade()` |
| `/mindmap` | Интерактивный граф цикла «источник → гипотеза → эксперимент → validation» с выбором ветки и Submit в разработку | Выбрать следующий цикл без ручного поиска контекста | `app/src/routes/mindmap/+page.svelte` |
| `/wallet/<address>` | Обновляемая карточка TON-кошелька: live TONAPI balance, holdings и ссылки на страницы монет | Исследовать конкретный smart-money wallet | `loadWallet()` + статические entries текущего roster |
| `/token/[addr]` | Полная карточка токена: график, держатели, MVRV, score, история сигналов, paper-трек | Глубокое погружение в один токен | `loadToken(addr)` (много источников) |

`/mindmap` — production control surface для ежедневного цикла. Выбор узла сохраняется локально до Submit; в Codex-контексте запрос отправляется follow-up сообщением, а на GitHub Pages открывается prefilled GitHub Issue (текст также копируется в clipboard). Это намеренно не выполняет код и не мутирует репозиторий напрямую.

Wallet routes строятся для двух закреплённых адресов Pavel Durov и текущего `data/wallets.json` roster. Данные страницы не замораживаются в build: после открытия holdings обновляются через TONAPI каждые 60 секунд.

## 4. Данные — словарь `data/*.json`

### 4.1. Базовые/справочные

| Файл | Содержимое |
|---|---|
| `index.json` | `{dates, tokens}` — список дат со снапшотами + addr→символ. Мастер-индекс. |
| `universe.json` | Широкий список всех обнаруженных токенов (для Screener/Compare). |
| `cats.json` / `cat_overrides.json` | Категория токена (meme/defi/stable/staking) + ручные правки. |
| `allowlist.json` / `blocklist.json` | Ручные форс-инклюды/эксклюды из отслеживаемого набора. |
| `funding.json` | addr → адрес, который первым профинансировал кошелёк (кэш для форензики). |

### 4.2. Рыночные снапшоты (ежедневный сбор)

| Файл / папка | Содержимое | Кто пишет |
|---|---|---|
| `snapshots/<date>.json` | Цена/держатели/supply/admin/пулы/TVL/объём/buys/sells на дату | `snapshot.py` |
| `intraday/<date>.json` | Второй срез дня (15:10 UTC) — удвоенное разрешение | `snapshot.py` (`INTRADAY=1`) |
| `alt/snapshots/<date>.json` | То же для CEX-монет вне TON (HYPE, ZEC) | `altsnapshot.py` |
| `holders.json` / `holders/<date>.json` | Концентрация держателей: `top10`, `top25`, `hhi` | `wallets.py` |
| `flows.json` / `flows/<date>.json` | Поток сделок: `trades_n`, `ubuyers`, `buyer_conc`, `buy_share` | `flows.py` |
| `forensics.json` / `forensics/<date>.json` | Бандл-детект: `bundle` (доля держателей из одного кластера финансирования), `clusters`, `max_cluster` | `forensics.py` |
| `social.json` / `social/<date>.json` | Упоминания $TICKER в TG-каналах: `mentions` | `social.py` |
| `health.json` | Свежесть и ошибки ключевых сборщиков; `/brief` предупреждает об устаревших данных | `health.py` |

### 4.3. Сигналы, скоринг, paper-боты

| Файл | Содержимое |
|---|---|
| `signals/<date>.json` + `signals/index.json` | Журнал сработавших торговых сигналов по дням |
| `signals/scores.json` | Статистика по типу сигнала: excess return, win rate, p-value, вердикт (`collecting`/`neutral`/`edge`) |
| `paper/bots.json` | Портфели `cons`/`aggr`: `cash`, `positions`, `trades`, `equity` (дневная кривая), плюс `wash_ban` (бан-лист токенов с накруткой объёма — на него опирается и деск) |
| `paper/altbots.json` | Портфель `alt` (CEX-монеты) в том же формате |
| `xs_forward_state.json` | Текущая открытая корзина long/short кросс-секционной momentum-стратегии |
| `xs_forward_equity.json` | Реализованный трек-рекорд (закрытые ротации) |
| `xs_audit.json` | Покрытие forward-трек-рекорда: legacy-строки, разложение long/short/fees и статус достаточности выборки |
| `dex_signals.json` | Разобранные посты из TG-канала `dexnewtoken` (buy/sell/новый листинг) |
| `perp_signals.json` | Разобранные алерты из `@perptools_ai_bot` (coin/side/entry/targets/stop/leverage) |
| `perp_markets.json` | Снапшот рынков Hyperliquid (фолбэк, когда API недоступен из браузера) |

### 4.4. AI Desk (`data/desk/`)

| Файл | Содержимое |
|---|---|
| `verdicts.json` | Главный выход деска: `{date, model, wallets:[{addr,name,manip_risk,flags,copy_ok,conviction,reason}], tokens:[{sym,manip_risk,flags,reason}]}` |
| `verdicts/<date>.json` | Дневная история вердиктов — на неё опирается `/brief` для диффа «что нового со вчера» |
| `calibration.json` | Самопроверка: правда ли высокий manip_risk предсказывает более слабую forward-доходность |
| `copytrade.json` | Point-in-time paper-проверка: весь roster vs `copy_ok` из verdict той же даты; при пустой книге Desk показывает только baseline/veto |
| `deployers.json` | Реестр деплоеров: `{deployers: {admin: {tokens, rugged, rug_rate}}, by_token}` — история рагов по создателю токена |
| `factors_active.json` / `factors_history.json` | Живые факторы риска, которые предложил и проверил агент-ресёрчер + аудит-лог всех попыток |
| `config.json` | Какой локальный LLM-бэкенд/модель использует деск |
| `worker_state.json` | Курсоры демона `desk_worker.py` (что и когда запускалось) |

## 5. Словарь сигналов и факторов — простым языком

### 5.1. Фичи деска (кошелёк/токен → 0..1, использует `desk.py`/`desk_features.py`)

| Фактор | Что измеряет |
|---|---|
| `wash` | Токен уже пойман на накрутке объёма (жёсткий флаг → `manip_risk: high`) |
| `co_entry` | Заходит ли кошелёк в токены синхронно с другими «умными» кошельками — признак координации/слива инсайда, а не собственного скилла |
| `vol_auth` | «Подлинность объёма»: высокое значение = похоже на органическую торговлю, низкое = вероятно накрутка/перекос buy/sell |
| `conc` | Концентрация держателей — сконцентрировано владение в нескольких руках или размазано |
| `edge_dispersion` | Заработал ли кошелёк свой edge на многих токенах (скилл) или на одном-двух (может быть везение) |
| `manip_risk` | Итоговый вердикт риска манипуляции (`low`/`med`/`high`) — детерминированный порог + LLM-обоснование поверх |
| `copy_ok` | Можно ли копировать кошелёк (false, если высокий риск или слишком мало сделок для доверия) |
| `conviction` | Уверенность LLM в вердикте `copy_ok` |

### 5.2. Временные ряды и рыночная механика

| Фактор | Что измеряет |
|---|---|
| `ret_1d`, `mom_3d`, `mom_7d` | Доходность за 1 день / моментум за 3 и 7 дней |
| `vol_z` | Z-score объёма — необычный ли сегодня объём относительно недавней нормы |
| `rvol` | Реализованная волатильность (насколько «дёргается» цена) |
| `trend` | Цена относительно своей 10-дневной скользящей средней |
| `hgrow_7d` | Рост числа держателей за 7 дней — органический интерес, а не только цена |
| `spread` | Разброс цены между пулами/DEX — признак неликвидности/арбитражной щели |
| `top_pool` | Крупнейший пул ликвидности токена |
| `age_d` | Возраст токена в днях — молодые токены рискованнее |
| `admin` | Адрес деплоера/владельца контракта |
| `top10`/`top25`/`hhi` | Доля предложения у топ-10/топ-25 держателей + индекс концентрации Херфиндаля |
| `ubuyers` | Число уникальных покупателей в выборке сделок |
| `buyer_conc` | Доля объёма покупок, пришедшая от горстки адресов — красный флаг накрутки |
| `buy_share` | Доля покупок в общем объёме (buy+sell) в долларах |
| `dep_rug` | Историческая доля рагнутых/забаненных токенов у этого деплоера — предупреждение ещё до появления рыночных сигналов |
| `bundle` | Доля держателей топ-25, чьи кошельки профинансированы одним источником — маскировка «распределённого» холдинга |
| `max_cluster` | Размер крупнейшего такого кластера |
| `mentions` | Число упоминаний $TICKER в отслеживаемых TG-каналах за день |

### 5.3. Торговые сигналы paper-ботов (`paper.py`/`altpaper.py`)

| Сигнал | Что означает |
|---|---|
| `hidden_buyer` | Много продаж, а цена не падает — кто-то тихо скупает предложение |
| `holders_surge` | Резкий рост числа держателей — новый органический интерес |
| `accum_div` | Цена падает, а держателей становится больше — накопление в слабость |
| `liq_inflow` | Ликвидность (TVL) быстро растёт при плоской цене — капитал заходит до движения цены |
| `momentum` | Обычный ценовой рывок (>8% за день) |
| `breakout` | Новый локальный хай на повышенном объёме |
| `dip_reversal` | Резкое падение у в целом здорового токена (много держателей/TVL) — кандидат на отскок |
| `flow_imbalance` | Активность сделок сильно смещена в сторону покупок |
| `vol_spike` (только alt) | Объём аномально высокий при плоской цене |

### 5.4. Вердикты score.py

`collecting` — данных пока мало · `neutral` — есть плюс, но статистически не значимо ·
`edge` — статистически значимая избыточная доходность (это и есть «сигнал реально
работает», после поправки на общее движение рынка).

## 6. Скрипты и расписание

Полный список — `scripts/*.py`. Каданс из `.github/workflows/`:

- **`snapshot.yml`** (ежедневно ~03:10 UTC): `universe → snapshot → paper → score →
  altsnapshot → altpaper → wallets → flows → social → dex_signals → forensics →
  xs_forward → xs_audit` — вся дневная цепочка сбора одним прогоном.
- **`intraday.yml`** (каждые 4ч): второй срез `snapshot.py --intraday`.
- **`perp-markets.yml`** (ежечасно): `perp_markets.py`.
- **`perp-signals.yml`** (каждые 4ч): `perp_signals.py`, `dex_signals.py`.
- **`deploy.yml`** (при пуше в `main`): билд и деплой сайта на GitHub Pages.

**Не в Actions — крутится 24/7 на локальном Mac** через `desk_worker.py`
(демон с governor'ом по питанию/термалке): `desk.py` (вердикты), `desk_calibration.py`,
`desk_copytrade.py`, `desk_deployers.py`, `desk_researcher.py`/`desk_factors.py`
(агент-ресёрчер факторов). Причина — локальная LLM (Osaurus/MLX на M1), Actions
такое не потянут. Гайд по сетапу — `docs/DESK-SETUP-m1.md`.

`signal_bot.py` и `xs_momentum.py` — офлайн-ресёрч, не подключены к живому пайплайну
данных (не пишут в `data/`, используются вручную для бэктестов).

## 7. Дневной срез (`/brief`)

Страница `app/src/routes/brief/+page.svelte` — единый экран того, что изменилось
за последние сутки, без нового сбора данных (только то, что уже лежит в `data/`):

- топ-5 роста/падения за 24ч по «ядру» рынка;
- деск: новые high-risk кошельки/токены, новые «можно копировать», топ по
  уверенности деска сегодня, свежая проверка копи-фида (`copytrade.json`);
- форензика/потоки: токены с высоким bundle-score, высокой концентрацией покупателей;
- соцсети: всплески упоминаний $TICKER день-к-дню;
- paper-боты: дневной P&L, momentum forward-test (последняя закрытая ротация +
  текущая открытая корзина);
- лента перп-сигналов (`@perptools_ai_bot`).

Все диффы «сегодня vs вчера» считаются на клиенте из уже собранных `data/*.json` и
их датированных версий (`data/desk/verdicts/<date>.json`, `data/flows/<date>.json` и т.д.).

## 8. Известные пробелы / направления (на 2026-07-07)

- Форензика транзакций на уровне блоков (настоящий детект бандлеров/снайперов по
  таймлайну трансферов) — не сделана, есть только proxy через кластеры финансирования.
  План — Phase 2 спека деска.
- PumpSense-подход (классификация постов TG-каналов локальной LLM) — отложено до
  появления трейдинг-каналов в конфиге.
- `signal-factory` (перп-сигналы через OKX, отдельно от деска) — есть спек
  (`docs/superpowers/specs/2026-06-26-signal-factory-design.md`), плана/кода нет,
  масштаб на порядок больше остального бэклога.
- `data/paper/state.json` и `data/desk/factors_active.json` (в некоторых состояниях
  репо) — проверять актуальность перед тем, как на них полагаться: первый похож на
  осиротевший файл, второй иногда отсутствует до первого прогона ресёрчера.

Полная история решений и обоснований — `ROADMAP-v3.md` (актуальный цикл) и
`ROADMAP-alpha-archive.md` (альфа, evolve-run #1–45).
