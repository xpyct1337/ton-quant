# TON Quant v3.0 — AI Smart-Money Desk (спек)

Дата: 2026-06-27 · Статус: утверждён к реализации (Igor, 27.06) · Версия: alpha → **v3.0**

## 1. Цель и тезис

Квантовый скачок v3.0: поверх готового пайплайна данных и сайта (v2) встаёт **AI Smart-Money Desk** — слой локальных LLM-агентов, который превращает наивный copy-trading-сигнал (`/wallets`: roster/conv/favorites) в **манипуляция-устойчивый, обоснованный фид**.

Обоснование (research, 27.06, см. `[[project_v3_research]]`): подход «нашёл прибыльный кошелёк → копируй» систематически эксплуатируется (бандлеры/снайперы-приманки, размазывание позиций, wash-объём/сентимент, слив в копир-трейдеров = «imitation penalty»). Лекарство, подтверждённое в литературе (arXiv 2601.08641): мультиагент-LLM + chain-of-thought + структурные анти-манипуляционные фичи → прибыльность кошелька становится предсказуемой.

Направление: **вглубь** (ширина «умеренно» — только то, что кормит деск). Монетизация отложена; `verdicts.json` проектируется как будущий продаваемый актив (оптциональность сохраняем).

## 2. Версия и «ресет прогресса»

## 3. Архитектура — три слоя, каждый делает одно

```
[Сбор]      GitHub Actions (ночью)  →  пишет data/*.json в репо   (как сейчас)
[Деск]      M1 Mac (launchd, ночью) →  читает data/, считает фичи,
                                        гоняет LLM-агентов, пишет data/desk/verdicts.json
[Витрина]   SvelteKit (статика)     →  читает verdicts.json, рисует риск+обоснование
```

Связь слоёв — только через файлы в репо (git). Сайт остаётся статикой, сервера нет.

## 4. Рантайм деска (M1 Air, 8 ГБ)

- **Ollama** (локально, Metal — MLX-бэкенд требует 32 ГБ, нам недоступен). Модель по умолчанию **`qwen3:4b`** (~4.5 ГБ — на 8 ГБ 4B надёжнее 7B, который свопит; сильное рассуждение + JSON; research 27.06). Альт `gemma3:4b` (100% JSON-parse, но стрипать болтовню). Env под 8 ГБ: `OLLAMA_FLASH_ATTENTION=1`, `OLLAMA_KV_CACHE_TYPE=q8_0`. Не брать Llama-3.2-3B (JSON-parse ~50%).
- Ограничение 8 ГБ → **тонкий LLM-слой:** вся структурная математика (фичи, статистика, PBO/DSR) — детерминированный Python; LLM читает компактный feature-JSON, не сырьё, и только судит/формулирует.
- Агенты идут **последовательно**, по одному в памяти (между агентами модель не дублируется). Без тяжёлых фреймворков (conytail) — Python дёргает локальный Ollama HTTP API (`/api/generate`, `format: json`) структурными промптами.
- Каденс: launchd ночью, `RunAtLoad=true` + `StartCalendarInterval` → catch-up при первом включении, если пропущено. Mac почти всегда включён.

## 5. Контракты данных

**Вход агентов** (детерминированный feature-JSON, считается Python из существующих `data/`):

```jsonc
// на каждый roster-кошелёк
{ "addr","name","conv","edge","win","ne","toks":[...],"new":[...],
  "wash": 0..1,            // wash-score держимых токенов (есть)
  "co_entry": 0..1,        // кластерность входа: доля токенов, куда кошелёк вошёл
                           //   одновременно с N др. roster-кошельками (из wallets/<date>.json диффов)
  "vol_auth": 0..1,        // аутентичность объёма (vol/TVL экстрим + buys/sells перекос)
  "conc": 0..1 }           // концентрация холдеров держимых токенов (top10/HHI)
```

**Выход** `data/desk/verdicts.json`:

```jsonc
{ "date","model",
  "wallets":[ { "addr","manip_risk":"low|med|high","flags":["bundler?","wash","conceal"],
                "copy_ok": true|false, "conviction": 0..1, "reason":"<1-2 предложения CoT>" } ],
  "tokens":[  { "sym","manip_risk","flags":[],"reason" } ] }
```

Гард витрины: нет `verdicts.json` / старая дата → `/wallets` рисует как сейчас, без риск-слоя (деградация).

## 6. Детерминированный слой фич (Python, MVP — только из имеющихся данных)

- `wash` — переиспользовать существующий Wash detector (объём против холдер/трейд-аутентичности).
- `co_entry` — из датированных `data/wallets/<date>.json`: если ≥K roster-кошельков впервые входят в один токен в один срез → координированный вход (прокси бандлинга/приманки).
- `vol_auth` — из snapshots: `vol24/TVL` экстрим + перекос `buys/sells` (накрутка ликвидности/объёма).
- `conc` — top10-доля / HHI держателей (есть в token-данных).
- `edge_dispersion` — `ne` и разброс: edge от одного токена (удача) vs многих (скилл) — уже считается.

Глубокая on-chain форензика (block-level бандлер/снайпер по таймлайну трансферов) требует нового фетча transfer-истории → **Phase 2**, не в MVP. conytail: MVP переиспользует то, что пайплайн уже собирает.

## 7. Агенты MVP

**Агент 1 — Манипуляция-аналитик.** Вход: feature-JSON токена/кошелька. Выход: `{manip_risk, flags, reason}`. Промпт: «оцени риск манипуляции по этим фичам; high если wash↑ И co_entry↑ И vol_auth↓…». Детерминированные пороги как floor, LLM — для синтеза и обоснования.

**Агент 2 — Вёттинг smart-money.** Вход: история кошелька + conv/edge/win/ne + вердикт агента 1. Выход: `{copy_ok, conviction, reason}`. Лекарство от imitation-penalty: `copy_ok=false` при `manip_risk=high` или тонкой выборке (ne малое при высоком edge).

Гейт: любой фактор, который позже предложит агент-ресёрчер (Phase 2), режется существующим **CPCV/PBO/DSR** — LLM его не обходит. Это наше преимущество против LLM-хайпа без дисциплины.

## 8. Витрина (`/wallets`)

- Значок риска у кошелька (`low`/`med`/`high`) + CoT-обоснование в тултипе.
- «Грязные» (`copy_ok=false`) — вниз/сворачиваются; чистые с высокой `conviction` — вверх.
- В favorites: токенам с высоким `manip_risk` — предупреждающий бейдж.
- `loadVerdicts()` в `data.js` (читает `data/desk/verdicts.json`, гард на отсутствие).

## 9. Фазы

- **MVP (Phase 1):** фичи (§6) + агенты 1+2 + витрина → манипуляция-устойчивый копи-фид. Шиппабельно.
- **Phase 2:** агент-ресёрчер альф (генерит/проверяет факторы под CPCV/PBO); глубокая on-chain форензика (transfer-таймлайн → настоящий бандлер/снайпер-детект).
- **Phase 3:** агент риск/портфель (сайзит paper-бот по вердиктам); срез Solana smart-money как эталон (ширина «умеренно»).
- **Позже:** монетизация (`verdicts.json` как продукт — подписка/SaaS/токен; решаем к 1.0/бете).

## 11. Сетап M1 (делает Igor, один раз)

1. Homebrew → `brew install ollama` → `ollama serve` (или login-item) → `ollama pull qwen3:4b` (+ `gemma3:4b` альт). Env: `OLLAMA_FLASH_ATTENTION=1`, `OLLAMA_KV_CACHE_TYPE=q8_0`.
2. `git clone` репо на Mac; git-креды для **push** (SSH-ключ или token) — деску нужно коммитить вердикты.
3. Python 3 (stdlib + `requests`); без тяжёлых зависимостей.
4. launchd-plist (по образцу `[[daily-brief]]`): `RunAtLoad` + ночной `StartCalendarInterval`.
5. Скрипт `scripts/desk.py`: pull → фичи → Ollama по агентам → `verdicts.json` → git add/commit/push.

## 12. Риски и как обходим

- **8 ГБ memory pressure** → 7B-Q4 + последовательные агенты + fallback 3B; при нужде лимит кошельков за ночь (топ-N roster).
- **Пассивное охлаждение Air** → ночной батч, кап токенов/контекста, не realtime.
- **Mac офлайн в момент крона** → launchd `RunAtLoad` catch-up.
- **Push с Mac** → git-креды на Mac (как deploy.bat на Windows); если Mac не пушит — деск пишет локально, Igor синхронизирует.
- **Маунт усекает запись** (Windows-сторона) — к Mac не относится; на Windows-правках сохраняем verify-by-Read.
