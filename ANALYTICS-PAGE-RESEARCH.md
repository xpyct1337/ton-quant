# TON Quant — страница «Аналитика»: ресёрч и план развития

*Ресёрч: 2026-06-23. Контекст: v1 (`index.html`, монолит) накопил ~12 аналитических карточек; v2 (`app/`, SvelteKit) при редизайне оставил только Treemap + Agents + Unlocks. Идея Игоря: собрать всё на отдельной странице «Аналитика» с переключателем «все дашборды / выборочно».*

---

## TL;DR

Аналитика v1 никуда не делась — она лежит в `index.html` и работает на «запечённых спарках» (40 дневных закрытий из снапшотов). **Главное открытие:** в v2 `loadAll()` уже отдаёт по каждому токену `row.hist` — ту же историю снапшотов, на которой строились все карточки. Значит порт ≈ перенос чистых функций (`pcorr`, `betaVs`, `rsiArr`, `macdArr`, `laggedCorr`…) в `$lib/metrics.js` + Svelte-обёртки. Новых запросов почти не требуется.

Рекомендация: не «копировать страницу целиком», а построить **реестр виджетов** — массив-манифест, где каждый дашборд это `{id, title, group, component, ready}`. Страница `/analytics` рендерит их по фильтру видимости, выбор хранится в `localStorage`, есть пресеты (профили) и «включить всё / только мои». Это стандартный паттерн конфигурируемого дашборда (Grafana library panels, Cloudscape configurable dashboard, Matomo widget palette) и он же решает запрос Игоря про тогглы.

Порядок внедрения: **(1)** каркас страницы + реестр на 3 уже-готовых виджетах → **(2)** допорт остальных карточек v1 → **(3)** 2–3 новых дашборда из бэклога на текущих данных → **(4)** платный слой.

## 1. Инвентарь: что было, что выжило, что портировать

### 1.1. Что было в v1 (`index.html`)

Двенадцать карточек, все — чистая клиентская математика, **0 новых запросов** (переиспользуют `spark[]`/снапшоты):

| # | Карточка (id) | Что показывает | Данные |
|---|---|---|---|
| 1 | Market map (`treemap`) | Карта рынка: размер = mcap, цвет = 24h | live rows |
| 2 | Correlation matrix (`corrCard`) | Тепловая матрица корреляций топ-12 + β к корзине | spark |
| 3 | Lead–lag radar (`llCard`) | Кто двигается первым, кто догоняет (лаг 1д) | spark |
| 4 | Momentum leaderboard (`momCard`) | RS-рейтинг + стрелка тренда (ускорение/торможение) | spark |
| 5 | Momentum oscillator / RSI (`rsiCard`) | Wilder-RSI + дивергенция (цена/RSI) | spark |
| 6 | Market breadth (`brCard`) | % токенов выше 7д-средней — ширина ралли | spark |
| 7 | TON Quant Index (`tqCard`) | Mcap-weighted индекс экосистемы | rows + spark |
| 8 | MACD trend (`macdCard`) | MACD-гистограмма + кроссовер + дивергенция | spark |
| 9 | Reversal watch (`revCard`) | Синтез RSI+MACD дивергенций (двойное подтверждение) | spark |
| 10 | Risk–return map (`rrCard`) | Доходность vs волатильность, точки по токенам | spark |
| 11 | Drawdown radar (`ddCard`) | Просадка от локального максимума | spark |
| 12 | Activity signals | Снапшот-дивергенции (rug_watch / accum / distrib / liq_inflow) | snapshots |

### 1.2. Что выжило в v2 (`app/`)

Только три блока на главной (`+page.svelte`): **Treemap** (= #1), **AgentCard** (агенты-сигналы), **Unlocks**, плюс **JettonTable**. Карточки #2–#11 в редизайн не попали. Это и есть «функционалы, которые убрали» — они существуют в коде v1, но не на новой странице.

### 1.3. Ключевой факт для порта

`data.js → loadAll()` уже строит по каждому токену:
- `row.hist` — массив снапшот-записей (price/holders/tvl/vol/buys/sells по дням),
- `row.growth` — рост холдеров.

Это ровно тот субстрат, что v1 называл `spark[]`. Поэтому **порт каждой карточки = (а) перенести чистую функцию-метрику в `metrics.js`, (б) написать тонкий `.svelte`-компонент-обёртку**. Математика уже валидирована (90+ unit-тестов в v1) — переносится 1:1, тесты тоже.

### 1.4. Приоритет порта (эффект / усилия)

1. **Correlation matrix + β** — вау-эффект, уникально для TON, чистый порт. ⭐ первой.
2. **Momentum leaderboard + RSI/MACD + Reversal watch** — связка моментума, ядро «что разворачивается». Переносятся пачкой (общий `spark`-pipeline).
3. **Risk–return map + Drawdown radar** — портфельный взгляд, дёшево.
4. **Market breadth + TQ Index** — индексные индикаторы «здоровья рынка», маленькие.
5. **Lead–lag radar** — надстройка над корреляцией, последней.

## 2. Новые дашборды из бэклога — готовность на текущих данных

Из `ANALYTICS-IDEAS.md`. Легенда: ✅ считается из данных, что уже приходят · 📸 нужен ряд снапшотов (у нас уже **12 дней**, копится) · 🔑 тяжёлый сбор (история по кошелькам).

**Готово прямо сейчас (✅) — брать первыми:**
- **Slippage Map** — из резервов пула (x·y=k) точный прайс-импакт ордера $100/$1K/$10K. Логика уже жила в `token-old.html` (`<h3>Slippage map</h3>`) — портировать на страницу-обзор по всем 20. Конвертирует «ликвидность» в торговое решение.
- **Wash-Trading Detector** — три флага: Volume/TVL > 3–5x при цене ±1%; volume/makers аномально высок; fee≠volume. Бейдж «🚿 wash?». Данные txns/tvl уже в снапшоте.
- **Volume×Price (абсорбция)** — buys≫sells при флэте = стена продавца; обратное = тихий выкуп. txns уже на руках.
- **Holder structure (шримпы/дельфины/киты)** — режем supply на корзины из топ-холдеров, стекед-бар. `StackedBar.svelte` уже есть в v2.
- **P/H ratio (mcap на холдера)** — кросс-скрининг переоценки, одна формула.
- **Спред-скринер всех 20** — кросс-DEX спреды уже считаются per-token, вывести таблицей.

**Дозревает на снапшотах (📸) — 12 дней уже есть, через 2 недели полноценно:**
- **Матрица «Цена × Холдеры»** (4 квадранта, тихая аккумуляция как buy-сигнал) — частично есть в Activity signals, развернуть в визуал.
- **Rug Radar (Ликвидность × Цена)** — TVL падает 3+ дня при стабильной цене. Сигнал `rug_watch` уже детектится — нужна история-вид.
- **CVD** — кумулятив (buys−sells) поверх цены.
- **Activity × Price (DAA-style)** — рост txns при флэте = интерес опережает цену.

**Тяжёлое (🔑) — позже, как «премиум-крючок»:**
- **Whale Tape** — единая лента крупных движений по топ-20 (нужен разбор транзакций кошельков). Самая залипательная фича, кандидат в платное.

**Вывод раздела:** на сегодняшних данных без единого нового источника можно поднять **6 ✅-дашбордов**. Это удваивает контент страницы поверх порта v1.

## 3. Монетизация / edge — что в платный слой

Принцип из памяти проекта: каждый цикл — деньги Игорю (80/20). Аналитика монетизируется по линии «бесплатно видно, что сигнал есть → платно видно, какой именно и что делать».

**Бесплатный слой (привлечение, ежедневный возврат):**
- Market map, индексы, breadth, корреляция — «обзорные», создают привычку заходить.
- Дивергенции/reversal watch — показываем, что *есть* сигнал, без точных тикеров в реалтайме.

**Платный слой (edge, за который платят):**
1. **Screener с комбо-фильтрами** (в духе Nansen, но бесплатно у конкурентов нет на TON): «Score>60 И холдеры↑7д И цена флэт И объём↑» → шорт-лист. Страница, ради которой возвращаются. — *highest-value.*
2. **Whale Tape (🔑)** — живая лента китов: «$48K UTYA → OKX». Залипательность + реальный edge. Платный, т.к. дорогой в сборе.
3. **Rug Radar + Wash-detector как алерты** — не «посмотреть», а «пуш в Telegram, когда TVL твоего токена −20%». Алерты = подписка.
4. **Slippage Map в исполнение** — «купить на $5K → двинешь цену 7.3%, лучше частями» — мост к боту/мини-аппу (TON Connect), будущая комиссия за исполнение.

**Канал:** Telegram Mini App (из ROADMAP) — вся аналитика + Alpha digest + алерты внутри Telegram, вход по TON Connect. Не источник edge, но сильнейший канал доставки для TON-аудитории и естественная точка оплаты.

**Что НЕ монетизировать в лоб:** обзорные индексы и treemap — это воронка, держим бесплатными.

## 4. UX тогглов и архитектура страницы в v2

Запрос: «включать все дашборды / на выбор». Это классический **configurable dashboard** — решается без тяжёлых библиотек.

### 4.1. Референсы (паттерны, не велосипед)

- **Grafana library panels** — виджет описан один раз, переиспользуется на любой странице; правка прокидывается во все инстансы. → у нас = реестр виджетов.
- **Cloudscape «configurable dashboard»** — палитра доступных айтемов (split-panel), показать/скрыть/добавить, reset-to-default.
- **Matomo / WordPress dashboard** — surface-agnostic registry типов виджетов + клиентский стор видимости.
- **Общий вывод индустрии:** разделять три вещи — **идентичность виджета (реестр)**, **видимость/layout (стейт)**, **persistence (localStorage)**. Не смешивать.

### 4.2. Предлагаемая архитектура (SvelteKit, минимализм)

**A. Реестр виджетов** — `app/src/lib/widgets.js`:
```js
// один манифест, single source of truth
export const WIDGETS = [
  { id:'corr',     title:'Correlation matrix', group:'Кросс-токен', ready:true,
    component:()=>import('./components/CorrMatrix.svelte') },
  { id:'leadlag',  title:'Lead–lag radar',     group:'Кросс-токен', ready:true,
    component:()=>import('./components/LeadLag.svelte') },
  { id:'rsi',      title:'RSI + дивергенция',   group:'Моментум',    ready:true, ... },
  { id:'rugradar', title:'Rug Radar',           group:'Риск',        ready:false /* 📸 */ },
  // ...
];
```
Группы: `Обзор · Моментум · Кросс-токен · Риск · Микроструктура`.

**B. Стейт видимости + persistence** — `$lib/dashboardPrefs.svelte.js` (руна `$state`), пишем в `localStorage` (как уже делает `token`-страница). Хранит `Set` включённых id + активный пресет.

**C. Страница `/analytics` (`routes/analytics/+page.svelte`):**
- сверху панель управления: **[Включить всё] [Свернуть всё]**, чипы-тогглы по группам, выпадашка пресетов;
- грид карточек (CSS grid, как в v1 — `auto-fill, minmax(320px,1fr)`), рендерим только включённые;
- **lazy mount**: `{#if visible.has(id)}<svelte:component>` + динамический `import()` — невидимый виджет не считает математику (важно, метрик много);
- данные грузим один раз в `+page.js`/`onMount` (`loadAll()` → `rows` c `hist`), прокидываем пропсом во все виджеты — каждый считает своё из общего `rows`.

**D. Пресеты (профили) — киллер-фича поверх «вкл/выкл»:**
- `Трейдер` = momentum + reversal + slippage + wash;
- `Инвестор` = corr + risk-return + drawdown + breadth + индексы;
- `Риск-менеджер` = rug radar + wash + holder structure;
- `Всё` / `Свой` (сохранённый набор).
Это и есть ответ на «все / на выбор» в продуктовой форме — пользователь не тыкает 18 галочек, а жмёт один профиль.

### 4.3. Почему так, а не drag-n-drop грид

`react-grid-layout`-стиль (перетаскивание/ресайз) — избыточен (conytail): добавляет зависимость и состояние ради фичи, которой на старте никто не просил. **MVP = вкл/выкл + пресеты + фиксированный responsive-грид.** Drag-n-drop — потом, если реально попросят. Видимость и layout разделены, так что доточить можно без переписывания.

### 4.4. Объём работ (оценка)

- Каркас страницы + реестр + persistence + пресеты: **~1 день.**
- Порт каждой карточки v1: **~0.5–1 ч** (функция уже есть, обёртка тонкая). 11 штук ≈ **1–1.5 дня.**
- 6 новых ✅-дашбордов: **2–3 дня.**
Итого работающая богатая «Аналитика» ≈ **неделя**, причём ценность инкрементальная с первого дня.
